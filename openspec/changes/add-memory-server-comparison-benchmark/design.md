# Design

## Fair-fight contract

Both contenders run as **persistent MCP servers over stdio**, on the same host, over the same
plain-markdown corpus generated by `scripts/synth_vault.py` (1k and 5k-note tiers; wikilink
density documented). Queries are a fixed set covering exact-title, keyword, and semantic
phrasings; the report publishes aggregates only (no query text, no paths — same privacy rule as
`src/exomem/eval_report.py`). Documented deltas rather than pretended equivalence: basic-memory
embeds with FastEmbed 384-d (semantic search optional), exomem with bge-base 768-d; write tools
compared are `write_note` vs `remember`; both sides' index/sidecar artifacts are built from
scratch in isolated config/home dirs.

basic-memory runs pinned (`uvx --from basic-memory==0.22.1 basic-memory mcp`) with the
safe-read knobs from the prior harness (`BASIC_MEMORY_DISABLE_PERMALINKS=true`,
`BASIC_MEMORY_ENSURE_FRONTMATTER_ON_SYNC=false`, `BASIC_MEMORY_SYNC_CHANGES=false`) so the
shared corpus files are never mutated; each contender gets an isolated `HOME`/config dir.

## Client

The harness talks MCP through the `fastmcp` client API (already a core dependency) with a
stdio transport per contender; per-call wall time via `time.perf_counter()`. RSS is read from
the spawned server process (psutil; if psutil is unavailable the RSS columns render as “—”).

## Measurement discipline

Warm-up calls excluded; median/p90 over N≥20 calls per query class; first-index time measured
once per tier (it dominates wall-clock); host recorded via `EXOMEM_BENCH_HARDWARE` convention.
The harness is desk-side (not CI): CI-speed variance would make cross-server deltas
meaningless.
